18 Words — <script> embed example

Instead of an <iframe>, the game can be dropped straight into a page with a single <script> tag. It mounts inside its own closed Shadow DOM tree, so its markup, CSS and element ids never leak into (or collide with) the host page — no cross-origin frame, no scrollbars to manage, no postMessage resize plumbing.

Prefer an iframe instead? See the iframe embed example.

Embed code

<div data-words18 data-lang="en"></div>
<script src="https://YOUR_DOMAIN/embed.js"></script>

data-lang accepts en, ru or th (defaults to en). You can place several [data-words18] containers on one page — each gets its own isolated widget instance, even with different languages. If you skip the container entirely, embed.js inserts one right where the <script> tag sits, using that tag's own data-lang attribute.

⚠️ Cross-origin requirement: the browser fetches style.css, index.html and days/*.json from the game's own domain, which needs CORS enabled (Access-Control-Allow-Origin) if that domain differs from the host page's. GitHub Pages sends this header by default; a custom static host may need it configured explicitly.
